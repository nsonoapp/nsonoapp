✅ CONCLUSION PRO NSONO

Ton intuition est correcte : dans cet ERP, les filtres finissent presque toujours avec createdAt DESC.

Mais après split expenses / debts / losses :
  ❌ plus de genre ASC, createdAt DESC  (collection expensess supprimée)
  ✅ remplacer par status / category ASC, createdAt DESC par collection

Priorité déploiement :
  0. role ASC, approvalStatus ASC (users — select admin entité, admin général)
  1. sellerId ASC, createdAt DESC          (stats.js + vendus.js — bloquant filtre vendeur)
  2. productId ASC, createdAt DESC         (sale_items vendus.js — bloquant filtre produit)
  3. payment_status ASC, createdAt DESC    (vendus.js — filtre paiement)
  4. status ASC, createdAt DESC            (vendus.js — filtre statut vente)
  5. sellerId + payment_status + status + createdAt DESC (vendus.js filtres combinés)
  6. amount_remaining DESC                 (stats.js — top dettes ouvertes)
  7. productId ASC, createdAt ASC          (FIFO stock_movements — perf)
  8. status ASC, DueDate ASC               (debts — échéances)
  9. le reste                              (finances, pertes, achats période)
